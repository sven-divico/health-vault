## Mission goal (v1 = "first version working")
Get a coherent first version that contains:
1. Image upload
2. Telegram-based activity logging
3. Web view of activity graph + uploaded images
4. Invitation / onboarding of a new user
5. Demo user + seed data (for visualization testing)

## Stated preferences
- **Visualization:** open-source libs; must support time-series zoom in/out, multiple
  visualization types, and pulling multiple data series into one view to see correlations.
- **UI pattern:** left scrollable icon sidebar; main sections expand to sub-sections;
  designed to keep expanding features while keeping a clean UX (admin-console pattern).

## Conclusions / decisions
- **Sidebar IA: by data domain** (Option A). Top-level sections each represent a kind of
  data; new tracked metrics become new sections. Sketch: Dashboard · Activity · Nutrition ·
  Body (Weight/Mood/Measurements) · Insights · Media · People (admin) · Settings.
- **Activity data model: light structure** (Option A). Activity entry = `category`
  (run/walk/gym/yoga…) + `durationMin` (number) + optional note. Bot keeps natural
  `/activity run 28min` style and parses the number. Enables minutes/day, sessions/day,
  per-category graphs, and correlation against mood/weight.

- **Charting foundation: Apache ECharts** (Option A). Replaces Recharts (to be removed
  from deps and from `MoodSparkline`/`MeasureCharts`). Chosen for native `dataZoom`
  (time-axis zoom/pan), many chart types, multi-series + multi-Y-axis, and brushing —
  maps 1:1 onto the zoom / multiple-viz / correlation requirements.

- **Insights / correlation view: stacked & synced by default** (Option B), with an
  **overlay (multi-Y-axis) toggle** (Option A) for 2-series comparisons. A series-picker
  (chips to add/remove metrics) feeds either mode. One shared `dataZoom` controls the
  time range across all lanes.

- **Demo user + seed data: rich & correlated** (Option A). ~90 days near-daily weight,
  mood, activity with an engineered storyline (activity ↑ → weight ↓ → mood ↑), ~15-20
  food entries incl. a few bundled CC0 meal photos. Demo user is created by the seed
  script, viewable on web via one-click "View demo" (no Telegram needed), and the seed is
  reseedable/isolated (wipe & regenerate without touching real users).

## Backlog (post-v1 / parking lot)
- **Smart input parsing & normalization:** general capability to convert free-form user
  input (Telegram text, captions) into structured numeric data across all entry types —
  activity, food, measures. AI-assisted; normalizes units, extracts metrics, disambiguates.
  v1 ships a simple parser for activity duration; this backlog item is the broader engine.
